あなたは今、こんな状態じゃないですか。

副業を始めたいけど、何から手をつけていいかわからない。
大学生で事業をやっているけど、売上が安定しない。
AIが稼げると聞くけど、自分には難しそうだと思っている。

この記事は、そんなあなたのために書いた。

俺は現役大学生で個人事業主だ。
物販と制作で累計売上1000万円を超えた。
でも同時に、売上256万を達成した翌月に売上0円になった経験もある。

成功だけ話すのは嘘になる。
だからこの記事では、失敗も全部話す。

読み終わったら、すぐ動ける。そういう内容にした。
最後まで読んでほしい。

- 現役大学生（今も授業に通っている）

- フォロワーは10人からのスタート

これが俺のリアルだ。

スーパーマンじゃない。
失敗もたくさんある。
でも、そこから学んで動き続けている。

この記事に書くことは、全部実体験からきている。

## 3. 売上256万を達成した話

大学1年の後半。
ある物販のジャンルで、思いがけず売上が爆発した。

月の前半だけで200万を超えた。
「これは本物だ」と思った。
友達には言わなかった。言えなかった。
自分でも信じられなかったから。

月末に合計すると256万円だった。

素直に嬉しかった。
でも同時に、ずっと不安だった。

なぜかわからないけど、「これは続かない」と感じていた。

その感覚は、正しかった。

## 4. 翌月、売上0円になった話（本題）

256万を売り上げた翌月の売上は、0円だった。

0円というのは、文字通り1円も売れなかったという意味じゃない。
費用を引いたら、手元に残ったのが0円だったということだ。

どうしてそうなったか。

最初は理由がわからなかった。
「市場が変わった」「競合が増えた」「ついていなかった」
そう思おうとした。

でも、違った。

一番の理由は、「なぜ売れたか」を分析していなかったことだ。

256万売れた時、俺は何もわかっていなかった。
たまたまタイミングが合った。
たまたまトレンドにのった。
たまたま検索に引っかかった。

「たまたま」で売れたのに、「再現性がある」と思い込んでいた。

これが、最大の失敗だった。

---

## 5. なぜ売れ続けられなかったのか｜3つの原因

失敗を振り返って、3つの原因がわかった。

### 原因①：成功を分析しなかった

売れた時、俺はその理由を記録しなかった。
どの商品が売れたのか。
どのタイミングで売れたのか。
どんな顧客が買ったのか。

何もわかっていなかった。

売れた経験は、分析しないと「運」で終わる。
分析して初めて「仕組み」になる。

### 原因②：仕組みを作っていなかった

256万売れた月、俺は毎日10時間以上働いていた。
それは「仕組み」じゃなくて「気合い」だ。

気合いは続かない。
体も精神も限界がある。

売れ続けるには、自分が動かなくても回る仕組みが必要だ。
俺にはそれがなかった。

### 原因③：次の手を考えていなかった

売れている時、次の手を考えていなかった。
「今売れているからいい」と思っていた。

でも市場は変わる。
競合は増える。
トレンドは終わる。

常に次の手を考えていないと、ある日突然終わる。
俺の場合、それが翌月だった。

---

## 6. AIと出会って変わったこと

0円の月を経験して、俺は変わった。

「頑張れば売れる」という考えを捨てた。
「仕組みを作ることが全て」という考えに変えた。

そこで出会ったのがAIだった。

最初はChatGPTの使い方がよくわからなかった。
「副業に使えるって聞いたけど、何に使えばいい？」という状態だった。

変わったのは、使い方を変えてからだ。

AIを「答えを出す道具」として使っていた。
それを「分析する道具」として使い始めた。

具体的に言うと、こうだ。

Before：「売れる商品タイトルを教えて」
After：「この10件の競合タイトルを分析して、共通するパターンを教えて。そのパターンを使った新しいタイトルを3案出して。」

この違いは大きい。

AIに「答え」を求めると、汎用的な答えが返ってくる。
AIに「分析」を求めると、具体的で使える答えが返ってくる。

この気づきが、全てを変えた。

---

## 7. 今実践している「売れ続ける仕組み」の作り方

0円の経験を経て、今俺が実践している仕組みを公開する。

### STEP1：売れた理由を毎回記録する

売れた時、必ずこれを記録する。

- どの商品・サービスが売れたか
- どのチャネル（X・note・物販サイト等）から来たか
- どんな人が買ったか（属性・悩み・きっかけ）
- 買う前にどんな行動をしていたか（投稿を読んだ、DMをした等）

これを記録するだけで、次に活かせるようになる。
「また同じことをやれば売れる」ではなく「なぜ売れたかがわかるから、別の商品でも同じパターンを使える」になる。

### STEP2：AIで分析の精度を上げる

記録したデータをAIに読み込ませる。
「このデータから、共通するパターンを教えて」と聞く。

人間が見逃していたパターンを、AIが見つけてくれることがある。
これだけで、次の一手の精度が上がる。

### STEP3：小さく試して、反応を見てから拡大する

256万売れた時、俺は最初から大きく動いた。
それは間違いだった。

今は「小さく試す→反応を見る→拡大する」の順番を徹底している。

投稿なら、まず1本出して反応を見る。
商品なら、まず少量で試してから在庫を積む。
サービスなら、まずモニターで試してから値段を決める。

失敗のダメージを小さくしながら、成功のパターンを積み上げていく。

### STEP4：再現性のある作業はAIに任せる

毎回同じ作業をしているなら、AIに任せることができる。

- リサーチ：競合調査・市場調査
- 文章作成：投稿・記事・商品説明文の骨格
- 分析：売上データ・反応データの傾向把握

これらをAIに任せることで、自分は「戦略」だけを考えられるようになる。

俺の場合、この仕組みを作ってから作業時間が約1/3になった。
そして、質は上がった。

## 8. 大学生が副業で稼ぐ上で絶対に知るべき3原則

大学生には、特有のリソースがある。
「時間」「失敗許容度」「好奇心」だ。

これを活かすための3原則を話す。

### 原則①：最初の1万円を稼ぐことだけを目標にする

「月収30万」「月収100万」という目標は、最初は意味がない。
まず1万円を稼ぐことに全力を使う。

理由は、最初の1万円を稼いだ時の感覚が全てを変えるからだ。
「自分は稼げる」という感覚を持った人間と、持っていない人間では、行動量が全然違う。

何でもいい。
物を売る、サービスを提供する、記事を書いて売る。
1万円を稼ぐ経験を、今すぐ作ることが最優先だ。

### 原則②：失敗を「コンテンツ」にする発想を持つ

大学生は失敗しても影響が小さい。
家族を養っていない。会社に属していない。
失敗してもやり直せる。

だからこそ、失敗を怖れずに動ける。
さらに言うと、失敗したことを発信すると、共感が集まる。

成功談より失敗談の方が、人は読みたがる。
これはリサーチで確認済みだ。

失敗は、コンテンツになる。
失敗を隠さない人間の方が、信頼される。

### 原則③：発信と事業を同時にやる

事業だけやると、「何をやっているかわからない人間」になる。
発信だけやると、「何も生み出していない人間」になる。

両方やる。

事業で実績を積みながら、それをリアルタイムで発信する。
失敗も成功も、全部コンテンツにする。

これが最速で信頼と実績を同時に積む方法だ。

## 9. X・noteでまだ収益化できていない俺が発信する理由

正直に言う。

俺はまだXもnoteも収益化できていない。
フォロワーは10人だ。

それでも発信する理由がある。

一つ目は、「過程」に価値があるからだ。
フォロワー0から1000、1万になる過程を見せられる人間は、今しかいない。
これは後から作れないコンテンツだ。

二つ目は、発信することで自分の思考が整理されるからだ。
「256万の翌月0円になった理由」を言語化することで、自分の中での解像度が上がる。

三つ目は、同じ状況の人に届けたいからだ。
完璧になってから発信しようとしたら、一生発信できない。
今の自分だから届けられる言葉がある。

だから発信する。

### ■ 物販×AI：商品リサーチから出品まで

**【プロセス全体像】**

```
市場調査（AI） → 競合分析（AI） → 商品選定（自分） → 
タイトル・説明文作成（AI） → 価格設定（自分） → 出品
```

**【具体的なプロンプト】**

Step1：市場調査
```
[ジャンル名]の市場で、今トレンドになっている商品を10件リストアップして。
各商品について：価格帯、ターゲット顧客、競合の多さ（少/中/多）を教えて。
```

Step2：競合タイトル分析
```
以下の10件の商品タイトルを分析して。
[タイトル1]
[タイトル2]
...
共通するキーワード、フックになっている言葉、数字の使い方のパターンを教えて。
```

Step3：自分の商品タイトル生成
```
以下のパターンを参考に、[自分の商品の説明]に合うタイトルを5案作って。
・パターン：[Step2で出たパターン]
・商品の特徴：[特徴を箇条書き]
・ターゲット：[どんな人が買うか]
```

このプロセスを使うと、タイトル作成の時間が2時間→20分になる。
質も上がる。

---

### ■ note×AI：記事テーマから執筆まで

**【プロセス全体像】**

```
テーマ調査（AI） → フック設計（AI） → 目次作成（AI） → 
本文執筆（自分＋AI） → 編集（自分） → 公開
```

**【具体的なプロンプト】**

Step1：テーマ調査
```
[自分のジャンル]でnoteで売れている記事テーマを10件教えて。
各テーマについて：なぜ売れると思うか、ターゲット読者、競合の多さを教えて。
```

Step2：フック設計
```
[テーマ]の記事を書く。
冒頭3行で読者の「痛み」に触れるフックを5パターン作って。
読者の痛み：[具体的な悩みを入力]
```

Step3：目次作成
```
[テーマ]の記事の目次を作って。
条件：
・3000〜5000字想定
・初心者でも理解できる構成
・読んだら即行動できる内容
・問題提起→原因→解決策→具体的手順→まとめの流れ
```

Step4：各見出しの本文を書く
```
[見出し名]の本文を書いて。
条件：
・一文は60文字以内
・具体的な数字を入れる（「たくさん」ではなく「月収30万」など）
・断定調と丁寧語を混ぜる
・箇条書きは5個以内
文字数：[300〜500]字
```

---

### ■ X運用×AI：投稿設計から分析まで

**【プロセス全体像】**

```
トレンド調査（AI） → フック設計（AI） → 投稿文作成（AI+自分） → 
投稿 → 反応分析（AI） → 次の投稿に反映
```

**【具体的なプロンプト】**

Step1：フック設計
```
[自分のジャンル]のXで伸びている投稿のフック（1文目）のパターンを分析して。
以下のパターンに当てはめて、[今日のテーマ]に合うフックを5つ作って。
・逆説型：「〜なのに〜」
・数字変化型：「〜から〜になった」
・全公開型：「正直に言います」
・常識破壊型：「〜は間違っている」
```

Step2：本文生成
```
以下のフックで始まるXの投稿文を作って。
フック：[選んだフック]
テーマ：[今日伝えたいこと]
文字数：200〜350文字
CTA：「保存しないと損します」or「フォローで毎日学べます」
```

Step3：反応分析
```
以下の投稿を分析して。
[投稿文]
いいね数：[数字]、RT数：[数字]、インプ：[数字]
・なぜこの数字になったと思うか
・改善するとしたらどこか
・次の投稿に活かすべき教訓を3つ教えて
```

### ■ 実際にこのプロセスを使って変わったこと

俺がこのプロセスを使い始めてから、変わったことがある。

**時間：** 1日の作業時間が約10時間→3時間になった
**質：** 投稿の反応率が平均2倍以上になった
